Alces Flight Storage Access Appliance

A simple, powerful interface to interact with your data in multiple storage locations, including POSIX filesystems and object storage

  • Learn More

Manage your data across multiple Alces compute environments

The Storage Access applicance provides you a single interface to manage multiple compute environments data, from POSIX storage on your ephemeral clusters to persistent data stored in your S3 buckets - the Storage Access appliance provides seamless interaction between POSIX filesystems and object storage systems. Using Alces Flight appliances in conjunction with the Storage Access appliance enables the appliance to dynamically add and remove members with no user-interaction required.

Dynamic Environment

Automatic registration and configuration

When used together with the supporting Alces Flight appliances, the Storage Access Manager appliance automatically deals with node registration and configuration. With registration and configuration taken care of for you - you can concentrate on research rather than setting up your environment.

Multiple storage platforms available

POSIX Storage

The Storage Access appliance supports POSIX storage types - in conjunction with the Alces Flight appliances, your cluster storage is dynamically added to the Storage Access appliance, allowing you to interact with multiple compute environments in a single interface.

Multiple storage platforms available

Object Storage targets can also be added to the Storage Access appliance, from compatible object storage interfaces such as Amazon S3 and Ceph RadOS gateways.
