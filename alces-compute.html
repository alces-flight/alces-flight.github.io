Familiar environment

The Alces Flight Compute appliance looks and feels like a traditional HPC cluster - backed by your favourite Cloud provider, allowing you to continue your research in the Cloud without needing to learn proprietary systems and techniques.

Dynamic Environment

Automatic registration and configuration

The Alces Flight Compute appliance automatically registers and configures compute hosts to the chosen cluster scheduler - nodes can be manually added to the environment and automatically join the environment, or optionally - Alces tools can deploy additional compute power for you when required.

Applications on-demand

Application Manager integration

An Alces Flight Compute environment used in conjunction with the Alces Flight Application Manager allows you to use your favourite applications again and again on each of your compute environments.

Stand-alone environment

An Alces Flight Compute environment can also exist in a stand-alone manager, providing Alces Gridware packages to the deployed hosts in your environment.

Why should I use an Alces Flight Compute cluster?

The Alces Flight Compute appliance has been designed to simplify the deployment of HPC compute environments in the Cloud, significantly reducing the time-to-research, as well as providing researchers with powerful tools to get started with their research in the Cloud
